# `i18n` — EN/ES bilingual subsystem

## Purpose

Single language toggle in the nav controls every translatable string on the landing page (40+ keys). All translatable elements carry a `data-i18n="key"` attribute (or `data-html="1"` if the value contains markup); the toggle calls `applyLang(lang)` which iterates and swaps `textContent` (or `innerHTML`). The rotor, marquee, doc tabs, and chat widgets each register a re-render hook (`window.fitRotor`, `window.renderActiveDoc`, `window.renderChat`) that runs after dictionary swap.

This is not a "component" you render — it's a page-level subsystem. It's documented here as a single .md so each component file can simply say "translates via i18n.md" and not re-explain the mechanism.

## Source

- HTML: `data-i18n` attributes throughout `Paperwork Monsters Landing.html` (45+ instances)

## Dictionary (verbatim — `Landing.html:5–108`)

The dictionary is a flat object keyed by `lang` then by `data-i18n` key. The values are **plain strings** unless `data-html="1"` is also on the element, in which case the value is HTML and gets `innerHTML`-injected (used for `<em>` highlights in section headlines).

## Toggle button (`Landing.html:1956–1959`)

```html
<div class="lang-toggle" role="tablist" aria-label="Language">
  <button class="on" type="button" data-lang="en">I speak English</button>
  <button type="button" data-lang="es">Yo hablo Español</button>
</div>
```

The button labels themselves are **deliberately bilingual** — "I speak English" / "Yo hablo Español" stay in their respective languages regardless of the active language. Don't run them through the dictionary.

## `applyLang()` (verbatim — `Landing.html:111–146`)

```js
let curLang = 'en';

function applyLang(lang) {
  curLang = lang;
  const dict = I18N[lang];

  // 1) plain text + html nodes
  document.querySelectorAll('[data-i18n]').forEach(el => {
    const k = el.getAttribute('data-i18n');
    if (dict[k] === undefined) return;
    if (el.getAttribute('data-html') === '1') el.innerHTML = dict[k];
    else                                       el.textContent = dict[k];
  });

  // 2) rotor words (data-en / data-es on each .word)
  document.querySelectorAll('.rotor-track .word').forEach(w => {
    w.textContent = w.getAttribute('data-' + lang) || w.textContent;
  });

  // 3) marquee — split pipe-delimited then re-insert with dot separators
  const mq = document.getElementById('marquee-track');
  const items = mq.querySelector('span').getAttribute('data-' + lang).split('|');
  mq.innerHTML = '';
  for (let i = 0; i < 2; i++) {
    const seg = document.createElement('span');
    seg.setAttribute('data-en', mq.dataset.en || '');
    items.forEach((item, j) => {
      const t = document.createTextNode(item);
      seg.appendChild(t);
      const dot = document.createElement('span');
      dot.className = 'dot' + (j % 2 === 0 ? '' : ' green');
      seg.appendChild(dot);
    });
    mq.appendChild(seg);
  }

  // 4) re-render docs section + chat in current language
  if (window.renderActiveDoc) window.renderActiveDoc();
  if (window.renderChat)      window.renderChat();

  // 5) resize the rotor track for the new word lengths
  if (window.fitRotor) window.fitRotor();
}

document.querySelectorAll('.lang-toggle button').forEach(btn => {
  btn.addEventListener('click', () => {
    document.querySelectorAll('.lang-toggle button').forEach(b => b.classList.remove('on'));
    btn.classList.add('on');
    applyLang(btn.dataset.lang);
  });
});
```

## Preact / Fresh translation

Drop the DOM-walking model. Use a Preact signal as the single source of truth, and have every component read from it.

```ts
// v2/frontend/lib/lang-signal.ts
import { signal } from "@preact/signals";

export type Lang = 'en' | 'es';
export const langSignal = signal<Lang>('en');
```

```tsx
// v2/frontend/islands/LangToggle.tsx — island
import { langSignal, type Lang } from "../lib/lang-signal.ts";

export function LangToggle() {
  const cur = langSignal.value;
  return (
    <div role="tablist" aria-label="Language"
         class="inline-flex items-center bg-white border border-coffee-500/16 rounded-full p-1 font-heading font-bold text-xs">
      {(['en','es'] as Lang[]).map(l => (
        <button type="button" role="tab" aria-selected={cur === l}
                onClick={() => (langSignal.value = l)}
                class={`px-3 py-1.5 rounded-full transition-all duration-200 ease-bounce
                        ${cur === l ? 'bg-teal-500 text-white' : 'bg-transparent text-teal-500/72'}`}>
          {l === 'en' ? 'I speak English' : 'Yo hablo Español'}
        </button>
      ))}
    </div>
  );
}
```

```ts
// v2/frontend/lib/i18n/landing.ts
export const LANDING_I18N = {
  en: { /* port verbatim from Landing.html */ },
  es: { /* port verbatim from Landing.html */ },
} as const;

export function t(key: string, lang: 'en'|'es'): string {
  return (LANDING_I18N as any)[lang][key] ?? key;
}
```

Each component reads `langSignal.value` and calls `t(key, lang)`. **No DOM walking** — Preact's reactivity handles the re-render automatically.

For the rotor / marquee / doc-tabs / chat re-renders that the legacy `applyLang` triggers manually, the Preact equivalents live in the respective component islands and are keyed off `langSignal` via `useEffect`.

## Server-side rendering note

For SSR, read the `Accept-Language` header in the route handler (Fresh `routes/index.tsx`) and seed `langSignal` to that value before render. Persist the user's choice to `localStorage` and read it on mount to avoid the FOUC of EN-then-ES.

```tsx
// routes/index.tsx
export const handler = (req: Request) => {
  const accept = req.headers.get('accept-language') ?? '';
  const initialLang = accept.startsWith('es') ? 'es' : 'en';
  return new Response(null, { headers: { 'X-Initial-Lang': initialLang } });
};
```

Then in the root layout, render `<script>window.__LANG__='es'</script>` before any island hydrates.

## Accessibility

- The `<button role="tab" aria-selected="…">` pattern is correct.
- Set `<html lang="…">` from `langSignal` so screen readers switch voice.
- The button labels stay in their own language ("I speak English" / "Yo hablo Español") — keep this; it improves discoverability.
- Don't use `aria-live` on the language toggle — the language change is intentional, no announcement needed.

## Edge cases

- **Mid-rotor lang switch:** the rotor measures the widest word in the *new* language and re-fits. Without re-fit, the cycling word would clip or jitter. The legacy `window.fitRotor()` and the Preact `useEffect` keyed on `lang` both handle this.
- **Mid-chat-reveal lang switch:** legacy clears + re-renders + restarts the IO. The Preact translation is keyed on `lang` so timers reset cleanly.
- **Missing key:** `dict[k] === undefined` short-circuits and leaves the existing text. The Preact `t()` returns the key itself so missing keys are visible during dev.
- **`data-html` XSS:** payload is from a hardcoded dictionary, never user input — safe. If the dictionary becomes CMS-driven, sanitize.
- **Localization of pricing-section pill (`::before { content: 'CON NOSOTROS' }`):** see `pricing.md` Edge Cases — replace pseudo-element with a real `<span>` that goes through the dictionary.
